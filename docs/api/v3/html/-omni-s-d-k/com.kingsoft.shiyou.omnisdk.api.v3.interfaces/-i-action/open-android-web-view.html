通过 AndroidWebView 打开网页(不再维护)

描述

  • 通过 AndroidWebView 打开网页。

  • 通过 Custom Tabs 机制 打开网页(默认)的机制原理，请查看 openBrowserActivity 的描述。

  • 通过 Custom Tabs 机制 打开网页，游戏进程不会被系统杀掉。

废弃说明

  • WebView objects allow you to display web content as part of your activity layout, but lack some of the features of fully-developed browsers.

    1. WebView 并不是浏览器，是用于加载通过系统WebView组件内核渲染后网页的内容容器。

    2. 根据手机系统版本、用户更新WebView组件版本的不同，WebView的渲染能力即 web 功能支持完整性是不一样的。

    3. OmniSDK 仅提供调用功能，并不对 WebView 进行深度开发使其具备手机浏览器的全功能特性。

    4. 因此，如果项目的网页一定要使用 WebView 或 Androd-JS 交互功能，则需要网页开发者自己完全适配不同手机系统的WebView版本支持度。

    5. 如果使用WebView，建议网页开发者不要在低版本手机系统使用web的高级特性；网页开发者需要对不同手机系统版本的WebView进行全覆盖的兼容性测试。

  • 注：OmniSDK 不对网页开发者在使用 WebView 进行加载网页上，出现的网页适配性问题进行技术支持，网页适配性问题属于web开发兼容性范畴。

  • 建议直接使用 openBrowserActivity 即 Custom Tabs 机制 打开网页(默认)。

webJson 数据结构

字段类型功能描述
web_urlString网页链接需要打开的网页链接
default_browserBoolean选择网页加载的容器true 通过 Custom Tabs 机制 打开网页(默认)；
false 通过 AndroidWebView 打开网页。
titleString页面标题页面标题
show_refreshBoolean刷新按钮true 显示；false 不显示
show_closeBoolean关闭按钮true 显示；false 不显示
show_prevBoolean上一页按钮true 显示；false 不显示
show_nextBoolean下一页按钮true 显示；false 不显示
layoutInt功能栏显示位置0 底部；1 顶部；2 不显示
orientationInt横坚屏1 强制横屏；2 强制坚屏；0 用户感应；
10 先横屏但允许用户感应；20 先坚屏但允许用户感应。
show_shareBoolean分享按钮通过SDK提供的分享接口进行分享：true 显示；false 不显示
share_paramShare-Json分享数据同 ISocial.socialShare 的 shareJson 数据结构及机制
game_headersJson-KVadd HTTP Request Headers添加一些 Headers
game_extString游戏透传内容网页动态通过 IJS.getCpParams 获取
custom_tabs_configJsoncustom tabs 容器可自定义功能查看 #custom_tabs_config 数据结构

custom_tabs_config 数据结构

  1. custom tabs 容器可自定义的功能；

  2. 系统默认浏览器的版本支持时支持，在某些低版本手机里可能无效果。

字段类型功能描述
show_titleBoolean网页标题是否显示网页标题：h5-title
close_iconres/drawable_id自定义关闭按钮icon24*24 PNG图片，res/drawable 文件夹下的图片id名
toolbar_color颜色值设置容器的Toolbar背景颜色格式：#AARRGGBB 或 #RRGGBB
