申请权限或权限组

申请权限或权限组

政策

  • 国内隐私权限政策规定玩家拒绝权限申请后，48小时不能重复弹窗申请，OmniSDK 内部处理。

  • 如果在政策时限内，游戏多次调用相同权限申请，直接返回拒绝。

描述

本接口申请权限的逻辑按照 Android 官方权限申请流程要求实现

  1. 首次申请权限时，与其他第三方实现无区别；

  2. 根据国家相关隐私政策，建议在需要权限的时候再申请。不要扩大范围申请，否则可能影响审核政策。

  3. **(在高版本)**首次被拒绝后，下次再申请前由系统显示rationaleContent的文案引导，用户同意才会触发权限申请；取消不触发，下次再重复这个流程。

  4. **(在高版本)**用户第二次拒绝后，再次申请可能不会触发权限申请，需要引导用户到权限管理界面申请权限。

接入说明

  1. 根据 Android 权限监听机制，权限结果回调只能在 Activity.onRequestPermissionsResult 收到。

  2. 为了简化游戏调用、注册步骤，游戏的主Activity 需要继承 OmniSdkActivity。

  3. 如果游戏接入 OmniSdkActivity，OmniSdkActivity 做好了 Activity.onRequestPermissionsResult 相关注册，游戏可在任意地方 使用registerPermissionsCallback 接收结果；

  4. 如果游戏没有接入 OmniSdkActivity，则需要参考 OmniSdkActivity 源码，进行相关的 Activity.onRequestPermissionsResult 注册。
