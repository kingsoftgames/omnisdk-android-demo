获取当前渠道账号体系的模式

  • 主要用在项目需要自定义账号登录UI界面和相关账号业务时候调用。

  • 由于OmniSDK为聚合类型SDK，在项目完成一次对接后需要保证在各个渠道平台正常发布；

  • 因此项目在自定义自身账号相关的业务时需要先调用该接口查询相应的账号体系模式；

  • 然后根据返回的账号登录体系模式结果来处理自定义账号UI的显示和相关业务逻辑处理。

登录

描述

  1. 若项目需自定义登录选项UI界面,请务必先调用接口getAccountMode获取账号模式，

  2. 然后根据返回的模式来进行自定义界面的显示和相关业务处理。

值类型

  • 指定登录账号类型时，key为"accountType"。

  • 0，表示不指定特定登录类型，直接使用默认登录界面；

  • 非0，表示指定特定登录类型(具体查看AccountType)，游戏使用自身的登录选项UI界面，这种方式游戏需要额外自己保存、更新登录信息；

游客账号

  • 接入的账号类型包含游客类型，绑定账号接口最好必接；

  • 游戏引导玩家主动绑定账号，防止帐号数据丢失。
