分享

分享方式

  1. 单个分享或项目自定义分享界面时，platform 传指定分享平台名。

  2. 多个分享（v1.0.9）：使用 OmniSDK 提供的分享弹窗UI界面，比如 platform="facebook,vk,system"

图片分享特殊说明

  1. 网络图片：需要项目先主动申请存储权限；可下载访问的有效链接。

  2. 截图：需要项目先主动申请存储权限

  3. 本地图片：因为Android7.0以上文件访问限制（非权限），需要转化本地链接（OmniSDK v1.0.9内部处理），否则高版本无法分享、甚至崩溃。

    • 请确保本地图片地址在Java方法，new File("local-path").exists() == true

    • OmniSDK 只处理了应用私有目录，项目需要其他存储目录请按FileProvider官方说明配置和相应不同手机的权限申请（不同厂商会细化权限）

    • 应用私有目录获取： activity.getExternalFilesDir(null).getPath()

    • v1.0.9 : imageUri = activity.getExternalFilesDir(null).getPath() + "/local_image_name.jpg" （转化由 OmniSDK 内部处理）

    • v1.0.9以下版本: 需要项目通过以下方式转化（建议升级最新版本）

          File image = new File(activity.getExternalFilesDir(null).getPath() + "/local_image_name.jpg");
             Uri localUri = FileProvider.getUriForFile(activity, activity.getPackageName() + ".FILE_PROVIDER", image); // 关键代码
             imageUri = localUri.toString()；
          val image = File(demoActivity.getExternalFilesDir(null)!!.path + "/local_image_name.jpg")
             val localUri = FileProvider.getUriForFile(activity, activity.getPackageName() + ".FILE_PROVIDER", image); // 关键代码
             imageUri = localUri.toString()；

platform 说明

  • 请使用OmniConstant.platform* 调用，防止参数名变化受影响。

  • "facebook" 通过Facebook-App分享：需要接Facebook-plugin，否则通过系统分享界面分享

  • "system" 通过系统分享界面分享

  • "jp.naver.line.android" 指定 line 应用（无原生分享方法）；如应用不存在，通过系统分享界面分享；v1.0.9 开始可以用 "line" 代替。

  • 游戏可以指定特定应用包名，如果用户安装了，直接分享；如果应用不存在，通过系统分享界面分享。

  • "com.whatsapp" 指定 whatsapp 应用；如应用不存在，通过系统分享界面分享

请注意 since

since 未说明的，请按系统或包名方式分享，否则不支持。

Since

  • 1.0 支持 facebook、system、指定包名

  • 1.0.8 支持 vk

  • 1.0.9 支持分享弹窗UI
