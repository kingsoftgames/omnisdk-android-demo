获取账号切换功能模式

该接口主要用在下面两种情况下:

  1. 项目在游戏内显示账号切换功能入口前务必先调用该接口进行判断，0关闭账号切换入口，其他值开启账号切换入口

  2. 若项目需要自定义账号切换UI的界面，此时务必先调用该接口然后通过返回的模式来完成相关账号切换UI界面的显示和业务处理

Return

账号切换功能模式:

  • 0 禁止提供账号切换的功能，项目需关闭账号切换的功能入口

  • 1 可以开启账号切换功能入口，但禁止自定义切换账号UI界面。触发切换账号时直接调用SDK的账号切换接口并保证accountType传值0

  • 2 可以开启账号切换功能入口，若项目自定义切换账号选项UI界面，则只能显示OmniSDK提供的账号类型选项。

  • 3 可以开启账号切换功能入口，若项目自定义切换账号选项UI界面，必须显示渠道账号，同时可选集成sdk提供的账号类型。
    对于渠道账号切换选项，项目需先调用OmniSDK提供的查询当前渠道信息的相关接口；然后通过返回的渠道数据（比如渠道ID，渠道名称等等）来定制渠道账号切换选项UI（比如背景图片，文字等等）的显示，
    并在触发渠道账号切换的时候调用SDK的切换账号接口并保证accountType传值-1
